# Page Mode Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Canvas Mode ↔ Page Mode 토글을 구현하여, 같은 요소 트리에서 자유 배치(Canvas)와 플로우 레이아웃(Page)을 전환할 수 있게 한다.

**Architecture:** DocumentStore에 `canvasMode`와 `pageViewport` 상태를 추가하고, 모드 전환 시 root의 layoutMode와 자식들의 position을 토글한다. Canvas Mode의 absolute 좌표는 `canvasPosition` 필드에 보존한다.

**Tech Stack:** MobX, React, TypeScript, postMessage bridge

- [ ] **Step 2: Add setCanvasMode method**

```typescript
setCanvasMode(mode: CanvasMode) {
  if (mode === this.canvasMode) return
  const root = this.elements.get(this.rootId)
  if (!root) return

  if (mode === 'page') {
    // Save absolute positions and switch to flow
    for (const childId of root.children) {
      const child = this.elements.get(childId)
      if (!child) continue
      child.canvasPosition = {
        left: typeof child.style.left === 'number' ? child.style.left : 0,
        top: typeof child.style.top === 'number' ? child.style.top : 0,
      }
      const { left, top, position, ...rest } = child.style
      child.style = { ...rest, position: 'relative' as const }
    }
    root.layoutMode = 'flex'
    root.layoutProps = {
      ...DEFAULT_LAYOUT_PROPS,
      direction: 'column',
      gap: 0,
      paddingTop: 0,
      paddingRight: 0,
      paddingBottom: 0,
      paddingLeft: 0,
      alignItems: 'stretch',
    }
    root.style = {
      ...root.style,
      width: this.pageViewport,
      height: 'auto' as unknown as number,
      minHeight: '100vh' as unknown as number,
      overflow: 'visible',
    }
  } else {
    // Restore absolute positions
    for (const childId of root.children) {
      const child = this.elements.get(childId)
      if (!child) continue
      const pos = child.canvasPosition ?? { left: 0, top: 0 }
      child.style = {
        ...child.style,
        position: 'absolute' as const,
        left: pos.left,
        top: pos.top,
      }
    }
    root.layoutMode = 'none'
    root.layoutProps = { ...DEFAULT_LAYOUT_PROPS }
    root.style = {
      ...root.style,
      width: this.viewport.width,
      height: this.viewport.height,
      minHeight: undefined,
      overflow: 'hidden',
    }
  }

  this.canvasMode = mode
}
```

- [ ] **Step 3: Add setPageViewport method**

```typescript
setPageViewport(width: PageViewportWidth) {
  this.pageViewport = width
  if (this.canvasMode === 'page') {
    const root = this.elements.get(this.rootId)
    if (root) {
      root.style = { ...root.style, width }
    }
  }
}
```

- [ ] **Step 3: Adjust viewport wrapper for Page Mode**

Page Mode에서는 root가 `width: pageViewport`, `height: auto`이므로 캔버스 배경에서 가운데 정렬:

```tsx
<div style={{
  transform: `translate(${panX}px, ${panY}px) scale(${zoom})`,
  transformOrigin: '0 0',
  position: 'absolute',
  ...(canvasMode === 'page' ? {
    left: '50%',
    marginLeft: -(pageViewport * zoom) / 2 / zoom,
  } : {}),
}}>
```

실제로는 viewport wrapper 안에서 정렬 방식을 조정. panX 기본값을 Page Mode 전환 시 센터로 리셋하는 것이 더 심플:

```typescript
// SET_CANVAS_MODE handler에서:
if (msg.payload.mode === 'page') {
  const el = outerRef.current
  if (el) {
    const rect = el.getBoundingClientRect()
    setViewport(prev => ({
      ...prev,
      panX: (rect.width - pageViewport * prev.zoom) / 2,
      panY: 20,
    }))
  }
}
```
